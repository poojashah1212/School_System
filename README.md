System Workflow

Teachers and students register and authenticate using JWT-based login.
Teachers can add and update student profiles associated with their account.
Teachers define weekly working availability, including days and time ranges.
Teachers configure holiday periods during which sessions cannot be scheduled.
Teachers create session slots strictly within defined availability.
The system enforces validations for holidays, working hours, and overlapping slots.
Session times are stored based on the teacher’s timezone.
Students log in to view available session slots.
Session times are presented in the student’s local timezone.
Students book available session slots.
Booked sessions are marked accordingly and visible to the teacher.
